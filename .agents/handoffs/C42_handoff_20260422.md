## Architecture — Data Flow

```
Firestore lead_assignments
  └─ db.js loadAssignedLeads()
       └─ batch CF calls (200/chunk, parallel) → masterLeadsById
       └─ fallback: assignment doc if CF fails
       └─ linkedInUrl normalized (https:// prepended)
            └─ PROSPECTS[] (global)
                 ├─ computeMetrics() [data.js]
                 │     └─ M.assigned / readyCount / enrichedCount / needsDataCount
                 │           └─ Command Center KPI strip
                 ├─ computeNicheMetrics() → Niche Performance cards
                 └─ pageLeadScoreboard() [pages.js]
                       ├─ isReady gate on "All" tab
                       ├─ ⏳ Needs Data tab
                       ├─ 200/page Prev/Next pagination
                       └─ Sortable column headers (window._sbSort)

Firestore meta/pipeline_stats
  └─ db.js fetchAdvisorLeadCount()
       └─ window._firestoreNicheBreakdown → Prospect Mine cohort cards
       └─ window._firestoreLeadTotal → computeMetrics().total
```
