## ✅ FIXES DEPLOYED THIS SESSION (all live in production)

### Fix 1 — App Check unenforced at project level
**Problem:** App Check was **Enforced** — 51% of auth requests (iPad, Firefox, DuckDuckGo) silently rejected.  
**Fix:** Firebase Console → App Check → Authentication → **Unenforce**.  
**Status:** ✅ Live — no code change needed.

### Fix 2 — iOS email autocapitalize
**Problem:** iPad keyboard capitalizes first letter → `Jsteward236` doesn't match Firebase account.  
**Fix:** `.toLowerCase()` on email input before `signInWithEmailAndPassword`.  
**File:** `js/auth.js` ~line 384

### Fix 3 — App Check skipped on iOS/iPadOS
**Problem:** reCAPTCHA Enterprise can't render on WebKit. Even in Monitoring mode, it blocked token acquisition on mobile.  
**Fix:** UA detection in `_initAppCheck()` — iOS devices skip the `firebase.appCheck().activate()` call.  
**File:** `js/auth.js` ~line 66–85

### Fix 4 — Firebase Functions SDK MISSING from index.html ⭐ ROOT CAUSE of 27-lead problem
**Problem:** `firebase-functions-compat.js` was never included. `firebase.functions()` threw on every load. The CF call silently failed, fallback path returned only 27 leads.  
**Fix:** Added `<script src="https://www.gstatic.com/firebasejs/9.23.0/firebase-functions-compat.js"></script>`  
**File:** `index.html` ~line 768

### Fix 5 — getLeadsByIds CF enforceAppCheck: true → false
**Problem:** CF rejected mobile clients without App Check tokens even after Fix 4.  
**Fix:** `onCall({ enforceAppCheck: false }, ...)` — CF still requires Firebase Auth.  
**File:** `functions/index.js` line 958  
**Deploy:** `firebase deploy --only functions:getLeadsByIds`

### Fix 6 — Email/phone object normalization (scoreboard blank)
**Problem:** PDL enrichment writes email as `{address: "...", type: "work"}`. CF returned raw master_leads data. `pages.js` called `.trim()` on an object → TypeError → scoreboard crashed blank.  
**Fix:** In db.js Step 3 mapping: `typeof v === 'object' ? (v.address || '') : String(v || '')`  
**File:** `js/db.js` ~line 243–246

### Fix 7 — pages.js .trim() safety guards
**Problem:** Secondary safety net for object-type fields.  
**Fix:** All `p.email.trim()` / `p.phone.trim()` guarded with `typeof === 'string'` checks.  
**File:** `js/pages.js` ~lines 497–498, 604–609
