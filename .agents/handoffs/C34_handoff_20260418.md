### C34-3: Agent A15 — Hennepin County Assessor GIS Miner

**File:** `scripts/agent_assessor_miner.js`  
**Source:** Hennepin County GIS ArcGIS REST API — `LAND_PROPERTY` MapServer Layer 1  
**Data:** Residential homestead parcels (`HMSTD_CD1 = 'H'`) with assessed market value ≥ $1M

**What it does:**
- Queries Hennepin County public parcel database (400K+ records, updated monthly, free, no key)
- Filters for: homestead classification (individual person, not LLC), property value threshold, target cities
- Default geography: 13 western suburbs (Wayzata, Minnetonka, Plymouth, Edina, Eden Prairie, Golden Valley, Hopkins, St. Louis Park, Orono, Long Lake, Deephaven, Shorewood, Chanhassen)
- Name parsing: `OWNER_NM` format is `FIRSTNAME [MIDDLE] LASTNAME` (verified from live API)
- LLC filter: skips records matching LLC/CORP/TRUST/FOUNDATION/HOLDINGS patterns
- Timing boost: +15 for 2024+ buyers, +8 for 2022+, +3 for 2020+
- Niche routing: `re-developers` for recent high-value buyers, `c-suite-executives` for all others

**First run results (150-lead batch):**
- 150 raw → 150 scrubbed (0 rejections — **perfect pass**)
- Avg property value: **$4,250,581**
- Cities: Wayzata (90), Minnetonka (24), Plymouth (25), Eden Prairie (10), Edina (1)
- 148 ingested (2 skipped — prior duplicates)
- Top lead: Michael Afremov, Wayzata — property value $16,042,600
- All HIGH confidence (≥0.80)

**Known caveats:**
- Trust/estate suffixes (`ET AL TRSTE`, `REV TR`) sometimes get parsed into the lastName field — scrubber catches most but a few slip through (e.g., `Michael Trste`, `Catherine Tr`). Enrichment will resolve.
- No phone/email in source — all records have `needsEnrichment: true`

#### 2. Insider Lead Name Resolution (5 leads)
The 5 ingested SEC Form 4 leads have `needsNameResolution: true` — their `firstName`/`lastName` fields are blank because the EDGAR API returns `rptOwnerName` in the Form 4 XML, not the structured fields.

**Resolution process:**
1. Pull each lead's `fecFilingUrl` / SEC form URL
2. Open the Form 4 XML and extract `rptOwnerName`
3. Update the Firestore `master_leads` record with the resolved name
4. Clear `needsNameResolution` flag

The 5 SEC submission API leads from the MN CIK whitelist DID get names (12 of 14 had names). The 5 blank ones were EFTS search results where the name wasn't in the structured field.
